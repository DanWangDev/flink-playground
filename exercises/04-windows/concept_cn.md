# 练习 04 — 窗口

## 你将学到

- **滚动窗口** — 固定大小，不重叠
- **滑动窗口** — 固定大小，重叠
- **会话窗口** — 基于活动，可变大小
- **ProcessWindowFunction** — 访问窗口元数据（开始、结束时间）

---

## 为什么这很重要

流是无限的。窗口将无限流切分为有限块以进行聚合。没有窗口，你无法计算"5 分钟平均值"或"每小时计数" — 它们是基于时间的分析的基础。

---

## 窗口类型

| 类型 | 大小 | 重叠 | 用例 |
|------|------|------|------|
| 滚动 | 固定 | 无 | "每 5 分钟，给我平均值" |
| 滑动 | 固定 | 有 | "每分钟，给我最近 5 分钟的数据" |
| 会话 | 可变 | 无 | "将间隔小于 2 分钟的事件分组" |
| 全局 | 整个流 | 不适用 | "计算所有事件" |

```
滚动 (size=3):
  [0-3)  [3-6)  [6-9)

滑动 (size=4, slide=2):
  [0-4)   [2-6)   [4-8)

会话 (gap=2):
  [1-5]    [8-12]
```

## ProcessWindowFunction

提供对窗口元数据的访问：

```java
.process(new ProcessWindowFunction<IN, OUT, KEY, TimeWindow>() {
    void process(KEY key, Context ctx, Iterable<IN> elements, Collector<OUT> out) {
        long start = ctx.window().getStart();
        long end = ctx.window().getEnd();
        // 聚合元素...
    }
})
```

## 你将练习

1. 使用**滚动窗口**（3秒）计算平均温度
2. 使用**滑动窗口**（4秒窗口，2秒滑动）计算重叠平均值
3. 使用**会话窗口**（2秒间隔）按活动分组事件

---

## 常见陷阱

1. **事件时间 vs 处理时间** — 此练习使用事件时间。如果没有设置水位线，窗口永远不会关闭。DataSources.sensors() 辅助函数会自动设置水位线。

2. **迟到事件** — 默认情况下，在水位线之后到达的事件会被丢弃。使用 `allowedLateness()` 让窗口保持更长时间打开。

3. **窗口状态** — Flink 将所有事件存储在窗口状态中，直到窗口触发。大窗口 = 大状态。

4. **使用 Duration，而非 Time** — 使用 `Duration.ofSeconds(n)` 代替已移除的 `Time.seconds(n)`。
